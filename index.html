Gherklin

An extensible linter for the gherkin syntax, powered by ESM and TypeScript

Get Started

Features

Extensible

Easily extend Gherklin by adding your own set of rules.

Easy to Use

Run either via the API or CLI and configure Gherklin with one file.

Reporting

With built in reporters, you can get your results whichever way you want them.

Examples

See the issues

The STDOUT reporter shows you all of the issues Gherklin saw while evaluating your feature files. It gives you a clear and concise view of the file, the severity of the issue, a short description of the issue and the rule.

Customize the rules

Gherklin allows you to customize each rule to meet your specific needs. You can turn rules off completely, set specific arguments for rules that allow them, and specify a severity for each rule.

Add new rules

If Gherklin doesn't quite hit your needs, you can extend it with custom rules. These custom rules allow the same fine tuned configuration as built in ones.

Installation

npm install gherklin

or

yarn add gherklin
